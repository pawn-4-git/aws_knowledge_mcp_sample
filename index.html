AWS WAF アプリケーションレイヤー (L7) DDoS 保護の概要

主な機能と特徴

AWS WAFは、主にHTTP/Sトラフィックを標的とするアプリケーションレイヤー（レイヤー7）DDoS攻撃を自動的に検出し、緩和するためのAntiDDoS AMR（AWSマネージドルールグループ）を導入しました。

  • 自動検出と緩和: 有効化後数分以内にトラフィックパターンを学習し、ベースラインを確立し、異常を検出して迅速（数秒以内）に攻撃を緩和します。
  • 攻撃タイプの識別: 悪意のあるDDoSイベントと正当な「フラッシュクラウド」（正当なトラフィックの急増）を区別します。
  • 緩和アクション: 主に2つのアクションを提供します：
    • チャレンジ: 正当なユーザーを確認するためにサイレントブラウザチャレンジを送信します。
    • ブロック: 悪意のあると特定されたリクエストを直接ブロックします。
  • 感度制御: BlockおよびChallengeアクションの両方で、検出された脅威に対するルールの感度（低、中、高）をカスタマイズできます。
  • 除外URI: チャレンジを処理できないURIの正規表現を指定でき、誤ってブロックされるのを防ぎます。
  • ラベル: 可視性を高め、カスタムルールを作成するために、リクエストにラベル（例: awswaf:managed:aws:anti-ddos:high-suspicion-ddos-request）を付与します。
  • 可視性: AWS WAFコンソール内のダッシュボードで、トラフィックソース、対象URI、緩和アクションを含むDDoSアクティビティに関するリアルタイムの洞察を提供します。詳細なログ記録と分析のためにCloudWatchと統合されています。

設定とベストプラクティス

  • 統合: AntiDDoS AMRをAWS WAF Web ACLにAWSマネージドルールグループとして追加します。
  • 優先度: 最大限の効果を得るために、AntiDDoS AMRルールグループをWeb ACL内で最も高い優先度（一番上、またはカスタムの「許可」ルールの直下）に配置し、できるだけ多くのトラフィックを検査できるようにします。
  • カスタマイズ: 特定のアプリケーションのニーズに合わせて保護を微調整するために、感度レベルを調整し、除外URIの正規表現を設定します。

利用可能性

AntiDDoS AMRは、以下のAWSサービスに関連付けられたAWS WAF Web ACLで利用可能です。

  • Amazon CloudFront
  • Application Load Balancer (ALB)
  • WAFでサポートされているその他のAWSサービス。

これは50 WCUを消費します。

結論

AntiDDoS AMRは、進化するレイヤー7 DDoS攻撃からWebアプリケーションを保護するための強力で自動化されたソリューションを提供します。WAFとShield Advancedの両方のお客様に、強化された可視性、カスタマイズ可能な制御、および柔軟な料金オプションを提供します。

詳細については、AWS WAF Managed Rules AntiDDoS AMRドキュメントを参照してください。
